概览

此页面是关于Substrate智能合约状态的概述。

虚拟机

FRAME provides two smart contract virtual machines which can be added to your Substrate runtime.

Contracts Module

The FRAME Contracts pallet (SEAL) provides functionality for the runtime to deploy and execute WebAssembly smart-contracts. It is designed to iterate on the design of modern smart contract platforms.

EVM 模块

The FRAME EVM pallet provides an EVM execution environment that allows unmodified EVM code to be executed in a Substrate-based blockchain. It is designed to most closely emulate the functionality of executing contracts on the Ethereum mainnet within the Substrate runtime.

自定义

Substrate 是一个完全没有任何条条框框限制的平台。

我们鼓励在 Substrate 运行时之上进一步开发其他智能合约平台。 使用这些预构建的模块来了解如何设计自己的系统或通过的现有系统来在 Substrate 链中工作。

智能合约语言

ink!

ink! is a Rust-based eDSL for writing Wasm smart contracts specifically for the Contracts module. It is designed for correctness, conciseness and efficiency.

智能合约与运行时模块的关系

Substrate 运行时模块和 Substrate 智能合约是使用 Substrate 框架来构建"去中心化应用"的两种不同途径。

智能合约

传统的智能合约平台允许用户在核心区块链逻辑之上发布额外的逻辑。 由于任何人（包括恶意参与者和缺乏经验的开发者）都可以发布智能合约逻辑，在这些公共智能合约平台上内建了许多刻意设计的安全守护手段。

以下是一些例子：

  • Fees: Ensuring that contract developers are charged for the computation and storage they force on the computers running their contract, and not allowed to abuse the block creators.
  • 沙箱: 合约无法直接修改核心区块链存储或其他合约的存储。 它的作用是限制合约只能修改自己的状态，且只能向其它合约或运行时函数发起外部调用。
  • 状态租赁: 合约占用了区块链上的空间，因此应该为其付费。 这确保人们不会从“免费、无限的存储”中获利。
  • 回退: 合约可能有导致逻辑错误的情况。 对合约开发者的期望很低，这就增加了额外的开销，以支持在交易失败时回退整个交易，这样当出错时不会更新状态。

这些不同的开销使得合约运行更慢、更昂贵，但是合约开发的“目标受众”与运行时开发者的不同。

合约可以让你的社区在你的运行时逻辑之上扩展和开发，无需各种令人发疯的提案、运行时升级，等等。 它甚至可以用作未来的运行时变更的测试基础，但可以将你的网络与可能发生的成长之痛或者错误进行隔离。

总之, Substrate 智能合约：

  • 对网络来说是天生安全的。
  • 通过经济激励来防止滥用。
  • 通过计算开销来支持优雅的逻辑错误处理。
  • 开发门槛更低。
  • 通过试验场使得社区快速互动以创建新的逻辑。

RunTime 模块

相反地，运行时模块不提供智能合约给你提供的这些保护或安全措施。 作为运行时开发者，你编写代码的门槛会非常高。

您可以完全控制网络上每个节点运行的基本逻辑。 您具有修改和控制所有模块的每一个存储条目的完整权限。 您甚至可以用错误的逻辑或糟糕的错误处理代码来堵死你的链。

Substrate 运行时模块开发的目的是制造精炼、高性能和快速的节点。 它不提供任何保护，不提供交易回退的开销， 也不隐式地引入区块链上节点运行的收费系统。 这意味着当您开发运行时功能时， 您 必须在运行时逻辑的各部分都正确地评估和应用手续费，才能使坏人无法滥用并伤害您的网络。

总之, Substrate 运行时模块：

  • 提供对整个区块链的底层访问权限。
  • 没有内置安全措施的性能开销。
  • 为开发者设置了高门槛。
    • 不一定要编写可工作的代码，但必须避免写有缺陷的代码。
  • 没有任何内生的经济激励机制来抵御坏人。

你的正确工具

Substrate 运行时模块和 Substrate 智能合约都是提供给您解决问题的工具。

在二者各自能够解决的问题种类方面可能有一些重叠之处，但是也有一些明确的情况只适合其中一个。 每个类别只举一个例子：

  • 运行时模块：在区块链中的交易之上建立一个隐私层。
  • 二者都可以：构建一个可能需要建立用户社区的游戏dapp (倾向使用智能合约)， 或者可能需要每天数百万交易的规模(倾向使用运行时模块)。
  • 智能合约：为您的区块链代币引入多重签名钱包。

除了上面写的所有内容，您还需要考虑到使用某个工具来计算创建一个 DApp 的成本。 部署合约是一个相对简单和容易的过程，因为你可以利用现有的网络。 唯一的成本是部署和维持合约而支付的费用。

相反，设立自己的区块链的成本还包括构建一个可以让人发现您的服务价值的社区，或者建立私有网络的云计算和日常网络维护的开销。

很难就每一种可能的情况提供直接指导，但使用下面的表格来帮助您进行决策，根据不同情形使用不同的工具：

运行时模块 二者均可 智能合约
  • Privacy Layer
  • Feeless Token
  • Light-Client Bridge
  • Decentralized Exchange
  • Oracles
  • Stable Coin
  • Game dApp
    • Small scale (contract)
    • Large scale (runtime)
  • Decentralized Autonomous Organizations (DAO)
    • Community driven (contract)
    • Protocol driven (runtime)
  • Treasury
    • Community driven (contract)
    • Protocol driven (runtime)
  • Multi-signature Wallet
