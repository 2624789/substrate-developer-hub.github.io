合约模块

The Contracts pallet provides the ability for the runtime to deploy and execute WebAssembly (Wasm) smart contracts.

Wasm 引擎

The Contracts pallet depends on a Wasm sandboxing interface defining the Wasm execution engine available within the runtime. 当前基于 Wasm 解释器 wasmi 实现。

功能

The Contracts module has a number of familiar and new features for the deployment and execution of smart contracts.

基于帐户

The Contracts module uses an account-based system similar to many existing smart contract platforms. 对于 Substrate 运行时，合约账户就像普通用户账户一样； 然而除了 账户ID 和 余额 之外，合约账户还关联了合约代码和持久化合约存储。

两步开发

Deploying a contract with the Contracts module takes two steps:

  1. 在区块链上存储 Wasm 合约。
  2. 用与 Wasm 合约相关的新存储实例化一个新帐户。

这意味着可以使用同一个 Wasm 代码初始化多个具有不同的构造参数的合约实例，从而减少区块链上 Contracts 模块所需的存储空间。

运行时环境类型

为了编写合约并与运行时交互，可以使用一组类型(例如 AccountId, Balance, Hash, Moment)。 可以使用自定义运行时中的自定义类型，也可以使用系统提供的默认类型。

合约调用

对合约的调用可以改变合约的储存、创建新合约，以及调用其他合约。 因为Substrate 为您提供了编写自定义运行时模块的能力，Contracts 模块也让您用合约账户直接异步调用 那些运行时函数。

沙盒化

The Contracts module is intended to be used by any user on a public network. 这意味着合约只能直接修改他们自己的存储。 为了提供给底层区块链的状态安全，Contracts 模块允许可逆交易，当合约调用未能成功完成时，回滚存储空间的任何更改。

燃料

合约调用需要支付 gas 费用，以限制交易可使用的计算资源数量。 在构造合约交易时，指定了燃料限额。 随着合约的执行，gas 根据计算的复杂性逐步被消耗。 如果在合约执行完成前达到 gas 限额，交易失败，合约存储被还原，gas 费用 并不 退还给用户。 如果合约执行完成时还有剩余 gas，则在交易结束时退还给用户。

Contracts 模块决定 gas 价格，这是Substrate Currency 与 gas 单位之间的换算。 Thus, to execute a transaction, a user must have a free balance of at least gas price * gas limit which can be spent.

存储租用

类似于 gas 限额对交易所使用的计算资源数量的限制，存储租金限制了合约在区块链存储中的占用。 合约账户按其使用的存储量所占的比例支付租金。 当合约余额低于规定限额时，合约账户会变成一个“墓碑”，其存储被清空。 当为墓碑合约提供了足够维持存活所需的资金，并提供了当时被清理的数据时，就可以恢复它。

Contracts 模块与 EVM 的关系

The Contracts module iterates on existing ideas in the smart contract ecosystem, particularly Ethereum and the EVM.

Contracts 模块和 EVM 之间最明显的区别是用于运行智能合约的基础执行引擎。 EVM理论上是很好的执行环境，但在使用现代硬件时并不实用。 For example, manipulation of 256 bit integers on modern architectures is significantly more complex than standard types. 甚至以太坊团队也在为下一代网络调研了 Wasm 的使用。

EVM只在储存发生时收取储存费。 这种一次性的成本导致区块链上的一些存储空间被永久占用，直到永远，这是很不经济的。 The Contracts module attempts to repair this through storage rent which ensures that any data that persists on the blockchain is appropriately charged for those resources.

The Contracts module chooses to approach contract creation using a two-step process, which fundamentally changes how contracts are stored on chain. Contract addresses, their storage, and balances are now separated from the underlying contract logic. This could enable behavior like what create2 provided to Ethereum or even enable repairable or upgradeable contracts on a Substrate based blockchain.
