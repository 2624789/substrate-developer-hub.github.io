Account Abstractions

Substrate 使用多组公钥/私钥对来表示网络的参与者。

概览

区块链系统有不同角色的参与者，从验证器到普通用户。

例如，Substrate 节点可以使用Nominated Proof-of-Stake (NPOS) 算法来选择 验证器。 验证者和提名者可能持有大量资金，所以Substrate's Staking 模块引入账户抽象，尽可能保障资金的安全。

这些抽象如下：

  • 帐户
    • 暂存秘钥（Stash Key）：暂存账户是为了持有大量资金。 Its private key should be as secure as possible in a cold wallet.
    • 控制器秘钥（Controller Key）：控制器账户代表背后的Stash 账户，例如用户的付费偏好，但只能持有最低限度的资金来支付交易费用。 Its private key should be secure as it can affect validator settings, but will be used somewhat regularly for validator maintenance.
  • 会话密钥（Session Keys）：会话密钥是保存在验证器客户端中的“热”密钥，用于对特定操作签名。 他们绝不应持有资金。

Note: Learn more about validators and nominators in the context of the Substrate's NPoS Staking module.

账户密钥

密钥对可以代表一个帐户并控制资金，就像您期望其他区块链会有 的普通帐户。 In the context of Substrate's Balances module, these accounts must have a minimum amount (an "existential deposit") to exist in storage.

帐户密钥是可以随意定义，在runtime才有实际意义。

在暂存账户和控制器账户例子中秘钥是以用途而区分的，在加密上没有区别。 从加密角度，创建的暂存秘钥或控制器密钥和所有普通账户密钥是一样的。

暂存密钥

Stash 密钥是定义Stash 帐户的公钥/私钥对。 这个账户就像一个 “储蓄账户”，因为您不应该经常从它进行交易。 其私匙应以最安全的方式处理，例如用一个安全 硬件加以保护。

由于Stash 密钥处于离线状态，它指定了一个控制器帐户以便做出一些与支付无关但和Stash 帐户资金权重相关的决定。 它还可以指定代理账户来代表它在 管理中投票。

控制器密钥

控制器密钥是定义控制器帐户的公钥/私钥对。 在 的Substrate的NPOS 模型中，控制器密钥会代表一个人进行验证或 提名。

控制器密钥用于设置首选项，例如在奖励中，为验证设置其会话密钥。 控制器账户只需要支付交易费用，所以 只需要少量资金。

绝不能使用控制器密钥从其Stash 账户中支用资金。 然而，控制器的行为可能导致花销，因此仍然应该有很好的保障。

会话密钥

会话密钥是“热密钥”，由验证器用于签署与共识相关的消息。 它们并不是用来作为控制资金的账户密钥，只应用于其预期用途。 它们可以定期更改；您的控制器只需通过向外广播公钥创建新的证书。 帐户密钥是可以随意定义，在runtime才有实际意义。

为创建会话密钥，验证操作者必须证明其拥有一个密钥能够代表其Stash帐户（抵押）和提名人。 为了做到这一点，他们使用自己的控制密钥对该秘钥签名，从而创建证书。 之后，他们在链上发布一笔包含该会话证书的交易，来通知区块链这一会话密钥代表了他们的控制密钥。

Substrate provides the Session module, which allows validators to manage their session keys.

强类型的封装器

你可以声明任意多的会话密钥。 例如，默认的 Substrate 节点使用 BABE, GRANDPA 和 "I'm Online"三个会后。 其他链可能有更多或更少，这取决于链需要验证器执行哪些操作。

这些不同的会话密钥使用相同的加密技术，但运行时的逻辑中的服务目的是完全不同的。 为了防止错误的操作使用错误的密钥，用Rust强 生类型封装着这些钥匙，使它们彼此不兼容，并确保它们只是 用于他们的预期目的。

生成

如果一个会话密钥被泄露，攻击者可提交导致节点权益损失的行为。 Session keys should be changed regularly (e.g. every session) via the rotate_keys RPC for increased security.
