SCALE 编解码器

SCALE（Simple Concatenated Aggregate Little-Endian）编解码器是一种轻量级，高效的二进制序列化和反序列化编解码器。

它被设计用在资源受限的执行上下文，例如 Substrate runtime，能够对数据进行高性能、无拷贝的编码和解码。 它不能进行自我描述，并假定解码上下文具有有关编码数据的所有类型信息。

Substrate 中的 SCALE

Substrate 使用 parity-scale-codec，这是 SCALE 编解码器的 Rust 实现。 该库和 SCALE 编解码器对 Substrate 和区块链系统是有益的，因为：

  • 它更轻量，通用序列化框架如 serde 增加了样板编码，使二机制文件过大。

  • 它不使用 Rust STD，因此可以将其编译为 Wasm，用在 Substrate runtime 之中。

  • 它对 Rust 的支持友好，能够为新的数据类型派生出编解码逻辑。

    #[derive(Encode, Decode)]
```

    为 Substrate 定义编码方案而不是重用现有的 Rust 编解码库是重要的，因为该编解码器需要在其它支持互操作性的平台和语言上重新实现。

    编解码定义

    在这里，你会了解 SCALE 编解码器如何对不同类型进行编码。

    定宽整数

    基本的整数类型使用固定宽度的小端（little-endian，LE）格式进行编码。

    示例

    • 带符号的8位整数69: 0x45
    • 无符号的16位整数42: 0x2a00
    • 无符号的32位整数 16777215: 0xffffff00

    紧凑型/通用整数

    “紧凑”或通用整数编码足以编码大整数（最大2 ** 536），并且比固定宽度版本更有效地编码大多数值。 (但是对于单字节数值，固定宽度整数永远是最好的。)

    编码结果的两个最低有效位用来表示模式：

    • 0b00：单字节模式；高六位是数值的 LE 编码（仅对数值0-63有效）。
    • 0b01：双字节模式：高六位以及后一个字节是该数值的 LE 编码（仅对值64-(2**14-1)有效）。
    • 0b10：四字节模式：高六位和随后的三个字节是值的 LE 编码（仅对值(2**14-1)-(2**30-1)有效)。
    • 0b11：大整数模式：高6位是其后的字节数减去4。 该值以 LE 编码形式包含于后面的字节中。 最后（最高有效）字节必须为非零。 仅对值 (2**30-1)-(2**536-1)有效。

    示例

    • 无符号整数0: 0x00
    • 无符号整数1: 0x04
    • 无符号整数42: 0xa8
    • 无符号整数69: 0x1501

    错误:

    • 0x0100: 用模式1编码数值0

    布尔

    布尔值使用单个字节的最低有效位进行编码。

    示例

    • boolian false: 0x00
    • boolian true: 0x01

    Options

    特定类型的单个值或没有值。 编码为：

    • 0x00 如果它是None ("空" 或 "null")。
    • 0x01 后面加上该值的编码，如果是Some的话。

    有一个例外，如果类型是布尔， 那么它总是一个字节：

    • 0x00 如果它是 None ("空" 或 "null")。
    • 0x01如果值是false。
    • 0x02如果值是true。

    Vectors (lists, series, sets)

    对相同类型的值的集合进行编码时，以条目数量的 compact 编码为前缀，然后依次为每个条目的编码，串联起来。

    示例

    无符号16位整数的Vector：

[4, 8, 15, 16, 23, 42]

<br />SCALE 编码后的字节：

0x18040008000f00100017002a00

<br />### 元组
固定大小的一组值，每一个值的类型可能不同，但是这些类型是预先确定和固定的。 这就变为每个值的编码的串联。 

#### 例子
紧凑的无符号整型和布尔型元组：
```
(3, false): 0x0c00
```

### 结构类型 
对于结构，将对值进行命名，但这与编码无关(名称将被忽略-仅顺序重要)。 **All containers store elements consecutively. 元素的顺序不是固定的，它取决于容器，并且不能在解码时依赖。
这隐含地意味着将某些字节数组解码为具有特定强制顺序结构，然后对其进行重新编码可能会导致与解码后的原始字节数组不同的字节数组。

#### Example
想象一个 `SortedVecAsc&ltu8&gt` 结构，它总是具有按升序排列的字节元素，而您拥有[3, 5, 2, 8]，其中第一个元素是其后的字节数（而 `[3, 5, 2]` 是无效的）。

`SortedVecAsc::from([3, 5, 2, 8])` 会解码为 `[3, 2, 5, 8]`，这与原始编码不匹配。

### 枚举 (tagged-unions)
固定数量的变体，每个变体互斥，并且潜在地隐含另一个值或一系列值

编码后的第一个字节表示该值所对应变体的索引。 其余的字节数是该变体所包含的实际值的编码结果。 
因此，最多支持256个变体。

#### 例子
```rust
enum IntOrBool {
  Int(u8),
  Bool(bool),
}
```
  • Int(42): 0x002a
  • Bool(true): 0x0101

实现版本

目前，Parity SCALE 编解码器的实现版本有：

  • Rust: paritytech/parity-scale-codec
  • Python: polkascan/py-scale-codec
  • Golang: ChainSafe/gossamer
  • C++: soramitsu/scale
  • JavaScript: polkadot-js/api
