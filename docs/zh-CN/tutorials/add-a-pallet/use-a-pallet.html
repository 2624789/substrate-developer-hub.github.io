与 Nicks 模块互动

现在，您可以编译并运行刚才通过 Nicks 模块增加的昵称功能的节点了。 用以下命令以发布模式编译节点：

WASM_BUILD_TOOLCHAIN=nightly-2020-10-05 cargo build --release

如果构建失败，请返回前面章节，并确保正确执行了所有步骤。 构建成功后，您可以启动节点：

# 在开发模式运行临时节点
./target/release/node-template --dev --tmp

启动前端

与之前的教程一样，本教程将使用 Substrate Developer Hub 里的 Front-End Template 来支持您与 Node Template 之间的交互。 只要已完成《创建您的第一条区块链》和《构建一个dApp》的教程，您就已经准备好，可继续本教程的其余部分了。

如有需要，请直接参考《创建您的第一条区块链》教程的 前端设置说明。

要启动 Front-End Template，导航到相关目录下，并运行：

yarn start

使用Nicks 模块

您应该已经熟悉如何使用前端模板 与 Substrate 模块进行交互 了。 在本节中，我们将使用 Nicks 模块来进一步说明如何使用前端模板与 FRAME pallets 进行交互。 我们还将学习更多关于如何使用前端模板通过 Sudo 模块来调用特权函数的内容。Sudo 模块默认是包含在 Node Template 里。 最后，您会学习到如何解析 FRAME pallet 触发出的不同类型的事件和错误。 那我们开始吧。首先，请在前端模板的帐户选择器里选择 Alice 的帐户，然后使用 Pallet Interactor 组件调用 nicks 模块中的 setName 可调用函数。 您可以选择任何名称，只要它的长度不少于 MinNickLength 且不超过您在上一步中配置的 MaxNickLength 的长度。

如上图所示，前端模板会报告可调用函数的状态，并让您能观察到 Nicks 模块和其他构成链 runtime 模块发出的 事件。 现在，使用 Pallet Interactor 的查询功能从 Nicks 模塊的 runtime 存储 中读取 Alice 的昵称值。

返回类型是一个包含两个值的元组：Alice 的十六进制编码的昵称和为保留昵称而从 Alice 的帐户中冻结的金额。 如果您在 Nicks 模块中查询 Bob 的昵称，您会看到返回 None 值。 这是因为 Bob 尚未调用 setName 可调用函数，也没有存入保留昵称所需的资金。

使用 "签名" 按钮来调用 killName 可调用函数，并使用 Bob 的帐户 ID 作为该函数的参数。 killName 函数必须由在上一节中使用 Nicks 模塊的 Trait 接口配置的 ForceOrigin 来调用。 您可能还记得我们将其配置为 FRAME 系统的 Root 来源类型。 Node Template 的 chain specification 文件是用于配置 Sudo 模块 让 Alice 可以访问此来源。 前端模板使您可以轻松地通过 Sudo 模块以使用 Root 来源调用函数，您只需使用 "SUDO" 按钮调用函数即可。 由于我们只使用了 "Signed" 按钮而不是 "SUDO" 按钮，因此该函数是由与 Alice 帐户关联的 Signed 来源 而不是 Root 来源 调用 。

您会注意到，即使函数已经成功调用，也会在 Events 窗格中看见 BadOrigin 错误。 这意味着 Alice 的帐户仍需为这次函数调用支付 费用，但却没有执行任何状态更改，这是因为 Nicks 模块遵循了重要的 先检验，后写入 原则。 现在，请使用 "SUDO" 按钮调用相同的函数，同时传入相同的参数。

Sudo 模块发出了一个 Sudid 事件 来通知网络参与者 Root 调用了一个函数。但是，您会注意到所包含的调用函数失败了，并显示 DispatchError 错误 (Sudo 模块的 sudo 函数 是那层在外面包裹着的调用函数)。 特别是，这是 DispatchError::Module 变量 的实例，该实例报告了两个元数据： 索引 号和 错误 号。 索引号与产生错误的模块有关；它对应于 construct_runtime! 宏中模块的 索引 位置。 错误 号与该模块的 Error 枚举类型中相关变量的索引相对应。 使用这些数字查找模块中的错误时，请记住 第一个 位置与索引是从 0 开始的。 在上面的屏幕截图中，索引 号是 9 (就是 第十 个模块)， 错误 号是 2 (就是 第三 个错误) 。 根据 Nicks 模块在您的 construct_runtime! 宏中的位置，索引 号的编号会有所不同。 无论 索引 号是什么，您都应该看到 错误号为 2，它对应于 Nick 模块的 Error 枚举类型的 第三 个变量，即 Unnamed 成员。 这不应感到奇怪，因为 Bob 尚未保留昵称。

您应该要确认 Alice 可以使用 "SUDO" 按钮来调用 killName 可调用函数，并能强制清除任何帐户的关联昵称 (包括她自己的帐户)，当然前提是关联昵称要实际存在。 你可能会想试试下面的这些方法：

  • 添加一个昵称，该昵称可以比Nick 模块的 Trait 配置里定义的 MinNickLength 短或者比 MaxNickLength 长。
  • 为 Bob 添加一个昵称，然后使用 Alice 的帐户及 "SUDO" 按钮强行移除 Bob 的昵称。 最后切换回 Bob 的帐户并调用 clearName 函数。

添加其它 FRAME Pallets

在本教程中，我们专门介绍了如何导入 Nicks 模块，但是正如开头所提到的，每个模块的导入方法都会有些许不同。 但请不要担心，您可以始终参考 Substrate 节点 runtime的示范 文件，该 runtime 几乎包含了所有核心的 FRAME pallets。

在 Substrate 节点 runtime 的 Cargo.toml 文件中，您将看到如何导入不同的模块，在 lib.rs 文件中，您会看到如何把每个模块添加到您的 runtime 里去。 通常您可以复制示例中的代码作为起点，以添模块到你自己的 runtime 里。
