可视化节点数据

Substrate显示指标的最新版本，就像你的节点连接了多少peers，你的节点使用了多少内存，等等。 通过这些可视化的指标，你可以使用像 Prometheus 和 Grafana一样的工具。

注意：在过去Substrate直接就暴露了Grafana JSON的端点。 但是现在已经被Prometheus指标端点替代了。

一个合理的架构看起来像这样：

      | `substrate_peers_count 5`        |                                                                          |
      |--------------------------------->|                                                                          |
      |                                  | --------------------------------------------------------------------\    |
      |                                  |-|在本地数据库中保存指标值与相应的时间戳 |    |
      |                                  | |-------------------------------------------------------------------|    |
      |                                  |                                         -------------------------------\ |

步骤 0：安装Prometheus 和 Grafana

  1. 安装 Prometheus 这里 2) 安装 Grafana 这里

步骤 1：运行你的节点

Substrate暴露一个在 Prometheus exposition format服务指标的端点在可用的9615端口上。 你可以使用--prometheus-port <PORT>更改端口，而并且允许它被接口以外访问而不是本地的--prometheus-external节点。

./substrate

步骤 2：检索指标

在第二个终端运行：

curl localhost:9615/metrics

可以返回一个相似的输出：

步骤 3：配置Prometheus去扫描你的Substrate节点

在 promeus.yml 配置文件中， 通过配置 Prometheus 将其添加到目标数组来扫描暴露的端点。

扫描配置：
  #这个工作名称从这个配置中作为一个标记`job=<job_name>`到任何的timeseries。
  - job_name: 'substrate_node'
    #每5秒钟从这个工作中覆盖全局默认值和扫描目标。
    scrape_interval: 5s

    static_configs:
      - targets: ['127.0.0.1:9615']

开始使用 prometheus.yml 配置文件准备一个 Prometheus 实例。

./prometheus --config.file prometheus.yml
