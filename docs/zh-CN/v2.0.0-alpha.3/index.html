Overview

此页面是 Substrate 文档的顶层入口。

Some Expertise Needed

为了充分利用 Substrate，你应该对区块链概念和基础密码学有很好的了解。 应该熟悉诸如区块头、区块、客户端、哈希、交易和签名之类的术语。 目前，你需要一定的 Rust 知识，才能对 Substrate 进行重要的定制化和适配（尽管最终，我们的目标并非如此）。

Substrate 是一个区块链开发框架，拥有一个完全通用的状态转换函数（STF），以及用于共识、网络和配置的模块化组件。

Despite being "completely generic", it comes with both standards and conventions - particularly with the Substrate runtime module library (a.k.a FRAME) - regarding the underlying data-structures that power the STF, thereby making rapid blockchain development a reality.

Usage

Substrate is designed to be used in one of three ways:

  1. With the Substrate Node: You can run the pre-designed Substrate Node and configure it with a genesis block that includes the default node runtime. In this case, you just need to configure a JSON file and launch your own blockchain. This affords you the least amount of customization, only allowing you to change the genesis parameters of the included runtime modules such as: balances, staking, block-period, fees, governance, etc... For a tutorial on doing this, see Start a Private Network with Substrate.

  2. With the Substrate FRAME: You can easily create your own custom blockchain using the FRAME. 这种方式为你提供了极大的自由度，来编写自己的区块链逻辑，允许你更改数据类型，从模块库中选择通用模块，还可以添加自定义的模块。 由于它是通过链上逻辑直接控制的，因此可以在不触及区块生成逻辑的情况下进行很多更改。 如果是这种情况，则可以使用现有的 Substrate 二进制文件进行区块生成和同步。 如果需要修改区块生成逻辑，则必须构建一个新的二进制文件，包含新的区块生成逻辑，然后交由验证人使用。 这就是 Polkadot 中继链的构建方式，并且在不久的将来，这一方式几乎可以满足所有需求。 有关此内容的教程，请参阅 创建你的第一个 Substrate 区块链。

  3. With the Substrate Core: The entire FRAME can be ignored, and the entire runtime can be designed and implemented from scratch. This could be done in any language that can target WebAssembly. 如果可以使 runtime 与 Substrate 节点的区块生成逻辑兼容，那么你可以简单地从 Wasm 对象构建一个新的创世区块，并使用现有的基于 Rust 的 Substrate 客户端启动你的链。 如果不兼容，那么你将需要更改客户端的区块生成逻辑，甚至可能会更改区块头和区块序列化格式。 就开发工作而言，这是迄今为止使用 Substrate 最为困难的方法，同时也为你提供了最高的自由度进行创新。
