Session Keys

验证人使用会话密钥对共识相关的信息签名。 SessionKeys 是一种泛化、可索引的类型，它在runtime代码中被具体化。

为创建会话密钥，验证人操作者必须证明其拥有一个密钥能够代表其Stash帐户（抵押）和提名人。 为了实现这一目标，他们通过Controller密钥对会话密钥进行签名，从而创建一个证书。 之后，通过发送一笔交易公布会话证书来通知链，这一会话密钥代表了他们的Controller密钥。

你可以声明任何数量的会话密钥。 例如，默认 Substrate节点使用了三个。 其他链可能有更多或更少，这取决于链需要验证人执行哪些操作。

pub struct SessionKeys {
    #[id(key_types::GRANDPA)]
    pub grandpa: GrandpaId,
    #[id(key_types::BABE)]
    pub babe: BabeId,
    #[id(key_types::IM_ONLINE)]
    pub im_online: ImOnlineId,
}

Note: This code is just an example of the Substrate node at the time of writing. 最新实现请参考runtime代码。

The default Substrate node implements Session keys in the Session module.

生成和使用

会话密钥是必须线上持有的“热”密钥。 它们不是用于帐户密钥。 如果一个会话密钥被泄露，攻击者可提交导致节点权益削减的行为。 Session keys may be changed regularly (e.g. every session) via RPC for increased security.
